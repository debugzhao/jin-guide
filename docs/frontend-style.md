# LiteLLM 官网视觉风格分析
https://www.litellm.ai/

> **已废弃（2026-07-12）**：本文档分析的深色风格曾在 v2.3 落地，用户实测反馈后已改回纯白背景浅色主题（`docs/frontend-prd-v2.md` v2.4 §3），本文仅作历史参考，不再是当前设计系统的依据。

基于当前 LiteLLM 官网观察，它好看不是因为"特效多"，而是因为它把一个复杂技术产品包装成了一个非常清楚的产品叙事。核心是：**少元素、高对比、强中心、产品架构可视化、可信背书、克制动画**。

## 布局

LiteLLM 首屏是很典型的 SaaS / developer tool 产品站结构：

### 顶部 announcement bar

一条浅紫色横幅，先给出动态消息，比如 "migrating to Rust"。它让页面一进来就有"产品在快速演进"的感觉。

### 悬浮导航条

不是传统满宽导航，而是一个深色圆角容器悬浮在页面上方。品牌、Hiring badge、菜单按钮都在一个玻璃感容器里，像产品控制台的一部分。

### 中心对齐 Hero

标题、说明、按钮全部居中。它没有左右分栏抢注意力，而是把用户视线直接压到主张上：

> Give Developers Azure Access

### 主视觉不是插画，而是架构图

中间是 LiteLLM gateway，周围是用户、模型供应商图标。这个特别关键，它不是"装饰图"，而是在解释产品工作方式。

### 往下是信任链路

客户 logo → 大数字指标 → What is LiteLLM → Features。

这个顺序非常产品化：先说我是谁，再证明有人用，再解释怎么用。

## 视觉

它的视觉高级感主要来自**控制**，而不是堆料。

### 深紫黑背景

背景不是纯黑，而是 `#040128` 附近的深紫黑。这样比普通黑色更有 AI / infra 气质。

### 低饱和紫色光晕

页面两侧有模糊光源，但很淡。它制造空间感，却不抢内容。

### 白色主标题 + 淡紫强调词

"Give Developers" 是白色，"Azure Access" 是淡紫色。这个颜色层级非常明确：白色负责确定性，紫色负责品牌记忆。

### 大面积留白

虽然是深色页面，但它没有把元素塞满。模块之间距离很大，所以显得贵、稳、可信。

### 细边框玻璃卡片

导航、gateway、按钮、视频卡片都是半透明深色 + 细白边。这种方式比蓝色发光边框更像真实产品，而不是展厅大屏。

### 字体很克制

没有中文海报式粗大标题堆满屏。标题大，但字重不炸；正文灰白，弱化解释文字。

## 交互

LiteLLM 的交互不是"炫技交互"，而是"产品理解交互"。

### 动态关键词切换

Hero 里的访问对象会轮播，比如 Azure / Gemini / Bedrock / OpenAI / Anthropic。这不是为了动而动，而是在告诉你：它接很多模型。

### CTA 很少

首屏主要两个按钮：Request Pricing、Deploy On-Prem。选择少，用户不会迷路。

### 菜单按钮极简

右上角一个圆角菜单按钮，强化"控制台 / app-like"的感觉。

### 架构图可读

中央 gateway 里面列 Cost Tracking、Model Access、Budgets、Rate Limiting、Prompt Management。用户不用读长文，就知道产品能力。

### Feature 区是渐进解释

先用大标题问 "What is LiteLLM?"，再给一个视频/demo，再进入 Features。它不是一上来甩 12 个功能卡片，而是一步步让用户进入。

## 动画

LiteLLM 的动画很克制，所以显得高级。

### 背景光晕慢速变化

不是粒子乱飞，而是低频、柔和、几乎不可察觉的动效。

### 标题关键词轮播

动的是"核心卖点"，不是装饰物。

### 架构图轻微发光

中央 gateway 有一点光感和景深，让它像一个真实系统节点。

### Logo / 内容出现节奏慢

滚动时模块出现比较自然，不会夸张弹跳。

### 没有过度 3D

它没有 Three.js 大场景，也没有复杂粒子。原因很简单：这个网站卖的是基础设施可信度，不是游戏或发布会视觉。

## 最重要的审美原则

LiteLLM 好看，本质是这几条：

1. **信息先于装饰**：每个视觉元素都在解释产品。
2. **一个屏幕只讲一个重点**：首屏讲 access，下一屏讲 trust，再下一屏讲 what，再讲 features。
3. **用产品界面当主视觉**：比抽象科技插画更可信。
4. **颜色少但记忆强**：深紫黑 + 白 + 淡紫，足够了。
5. **动画服务叙事**：动的是模型访问对象、数据流、产品模块，而不是无意义炫光。
6. **像开发者工具，不像政府/企业宣传展板**：这就是高级感的分水岭。
